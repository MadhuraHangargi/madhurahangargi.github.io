# ContractManagement
- A simple frontend-based contract management system built using HTML, CSS, and JavaScript. It allows users to create contracts, track their status, and manage the full contract lifecycle from a dashboard with a backend.

#FEATURES:
-Creates contracts using predefined templates
-Live contract preview while filling details
-Contract lifecycle tracking: Created → Approved → Sent → Signed → Locked
-Dashboard with real-time status counts
-Floating AI Assistant available on all pages
-We can Download signed contracts as PDF
-Data stored using LocalStorage (no backend)

#Tech Stack:
1.HTML5
2.CSS3
3.Vanilla JavaScript
4.LocalStorage
5.jsPDF & html2canvas

#FILE INFO
1.index.html
-This is the login page of the application where the user enters details and gets redirected to the dashboard after login.

2.dashboard.html
-This is the main dashboard that displays the list of contracts, contract phase counts, logged-in user info, and the floating chatbot.

3.create-contract.html
-This page is used to create a new contract by selecting a template, filling required details, previewing the agreement, and saving it.

4.view-contract.html
-This page shows a single contract in detail along with its timeline actions like approve, send, sign, lock, and download as PDF.

5.blueprints.html
-This page displays all available contract templates and allows the user to choose one while creating a new contract.

6.css/style.css
-This file contains all the main styling for the application including layout, sidebar, tables, buttons, and cards.

7.assets/chatbot/chatbot.html
-This file contains the HTML structure of the floating chatbot that appears on all pages.

8.assets/chatbot/chatbot.css
-This file handles the chatbot’s design, positioning, animations, and visual appearance.

9.assets/chatbot/chatbot.js
-This file controls the chatbot behavior such as opening, closing, sending messages, and showing contract summaries.

10.readme.md
-This file explains the project, its flow, and how different files work together.
